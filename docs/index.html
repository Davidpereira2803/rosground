Monitor ROS topics on your phone — fast.

ROS Monitor is a minimal mobile client that lets you subscribe to topics via rosbridge and visualize messages (std_msgs, geometry, sensor feeds, and more). Optional HTTP video streaming lets you preview camera topics. Avoid using an enourmous amount of terminals to visualize data—use your phone instead!

Features

Topic Subscribe

Connect to rosbridge_websocket and subscribe to one or more topics.

Message Viewer

Pretty JSON view with type badges, rate, and last updated time.

Camera Preview

Optional HTTP MJPEG server to view camera feeds with low latency.

ROS1 & ROS2

Works with ROS 1 (rosbridge_suite) and ROS 2 via rosbridge.

Quick Start

1
Install rosbridge 
# ROS 2 (Debian-based)
sudo apt install ros-${ROS_DISTRO}-rosbridge-suite
2
Run rosbridge websocket 
ros2 launch rosbridge_server rosbridge_websocket.launch.py
3
(Optional) Start video HTTP server

Download the file video_http_server.py from GitHub.

python3 video_http_server.py --topic /camera/image_raw --port 8080
Adjust --topic and --port as needed.
4
Connect from the app

Enter your machine's IP and port (default 9090) in the app settings, pick topics, and you're set.

Example network

Phone and ROS machine must be on the same network (e.g., Wi-Fi). If using a firewall, allow inbound 9090 (websocket) and your video port (e.g., 8080).

# find host IP
ip a | grep inet   # Linux
ipconfig           # Windows

# basic check from phone
http://YOUR_HOST:9090/

How to Use

  1. Ensure rosbridge websocket is running on your ROS machine.
  2. (Optional) Start the video HTTP server for camera topics.
  3. Make sure your phone and ROS machine are on the same network.
  4. Get your IP-Address.
  5. Open the app and add a connection (ws://IP-Address:9090).
  6. Browse available topics, choose ones to monitor, and save as a profile.
  7. Tap a topic to open the live viewer; for camera topics, open the Video tab.

FAQ

Which message types are supported?

Common std_msgs, geometry, sensor messages render as JSON. Custom messages display raw JSON.
